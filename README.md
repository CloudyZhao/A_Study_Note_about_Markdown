## 1. 标题

标题可以有两种方式来定义，具体方式如下：

### 1.1. 使用符号```#```来定义一~六级标题 【推荐用法】

Markdown中可以使用符号```#```来定义1~6级标题，放在标题的同一行最前方（三个空格之内），且符号```#```与文字之间必须间隔一个以上的空格，具体方法如下：

#### 1.1.1. 示例代码
```
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题

    # 一级标题【错误示例：符号#前面的空格为4个（不满足小于等于3个）】
   # 一级标题【正确示例：符号#前面的空格为3个（满足小于等于3个）】
   ##二级标题【错误示例：符号#前面的空格为3个（满足小于等于3个），但符号与文字之间没有空格（不满足大于等于1个）】
  ## 二级标题【正确示例：符号#前面的空格为2个（满足小于等于3个），且符号与文字之间存在1个空格（满足大于等于1个）】
 ###     三级标题【正确示例：符号#前面的空格为1个（满足小于等于3个），且符号与文字之间存在5个空格（满足大于等于1个）】
```

#### 1.1.2. 实现效果

> # 一级标题
> ## 二级标题
> ### 三级标题
> #### 四级标题
> ##### 五级标题
> ###### 六级标题
>
>     # 一级标题【错误示例：符号#前面的空格为4个（不满足小于等于3个）】
>    # 一级标题【正确示例：符号#前面的空格为3个（满足小于等于3个）】
>    ##二级标题【错误示例：符号#前面的空格为3个（满足小于等于3个），但符号与文字之间没有空格> （不满足大于等于1个）】
>   ## 二级标题【正确示例：符号#前面的空格为2个（满足小于等于3个），且符号与文字之间存在1个空格（满足大于等于1个）】
>  ### 三级标题【正确示例：符号#前面的空格为1个（满足小于等于3个），且符号与文字之间存在5个空格（满足大于等于1个）】

### 1.2. 使用符号```=```和```-```来定义一级和二级标题

Markdown中可以使用符号```=```来定义一级标题，使用符号```-```来定义二级标题，放在标题文字的下一行最前方（三个空格之内）。

**注意：**

**1. 【方法局限】此种方法中只能定义一级和二级标题，对于三~六级无法定义，因此不推荐此法。**

**2. 【格式要求】使用该语法定义标题时的符号```=```或```-```至少大于等于一个，不过即使有无数个也不影响实际效果。**

#### 1.2.1. 示例代码

```
一级标题
=

一级标题
  =========

二级标题
-

二级标题
   -------------------

一级标题
    =  【错误示例：符号=前面的空格为4个（不满足小于等于3个）】
```

#### 1.2.2. 实现效果


> 一级标题
> =
>
> 一级标题
>   =========
>
> 二级标题
> -
> 
> 二级标题
>    -------------------
>
> 一级标题
>     =  【错误示例：符号=前面的空格为4个（不满足小于等于3个）】

## 3. 强调文字

此部分主要对文字增加特殊的强调效果，比如斜体、粗体、添加删除线、添加背景色等。此处的文字效果可以应用于标题，亦可相互叠加强调效果。该部分有两个注意点，所有的强调语法都通用。

**注意：**

**1. 【格式要求】符号与待强调文字连接之处不允许有空格，但是待强调文字内部允许有空格。在段落中想要对文字进行强调，最好将强调文字与其他常规文字用空格隔开，具体对比效果如下所示：**

示例代码：
```
这处是 **_~~删除线＋粗体＋斜体~~_** 的文字
这处是**_~~删除线＋粗体＋斜体~~_**的文字
```

实现效果：
> 这处是 **_~~删除线＋粗体＋斜体~~_** 的文字  【强调文字与常规文字用空格隔开】
>
> 这处是**_~~删除线＋粗体＋斜体~~_**的文字  【强调文字与常规文字没有空格隔开】

**2. 【GFM与标准MD的不同点①】在GFM(GitHub Flavored Markdown)的语法中，文字内部下划线```_```和波浪号```~```不起强调作用，即把文字内的下划线```_```和波浪号```~```当成常规文字处理。因此，此处更推荐使用星号```*```来强调文字，或者先用星号```*```包围下划线```_```和波浪号```~```。**

### 3.1. 斜体文字

斜体文字是用一对星号```*```或者一对下划线```_```将待斜体的文字包围来定义。**注意：星号```*```和下划线```_```不能混用，必须一一对应。**

### 3.1.1. 示例代码

```
*斜体文字*
_斜体文字_
*斜体 文字*  【正确示例，符号与文字连接部分无空格，文字内部允许有空格】
* 不是斜体文字*  【错误示例】
*不是斜体文字 *  【错误示例】
这处是*斜体文字*
这处是_斜体文字_【GFM与标准MD的不同点①】
```

### 3.1.2. 实现效果

> *斜体文字*
>
> _斜体文字_
>
> *斜体 文字*  【正确示例，符号与文字连接部分无空格，文字内部允许有空格】
>
> * 不是斜体文字*  【错误示例】
>
> *不是斜体文字 *  【错误示例】
>
> 这处是*斜体文字*
>
> 这处是_斜体文字吗_【GFM与标准MD的不同点①】

### 3.2. 粗体文字

粗体文字用两对星号```*```或者两对下划线```_```将待粗体的文字包围来定义。**注意：星号```*```和下划线```_```不能混用，必须一一对应。**

### 3.2.1. 示例代码

```
**粗体文字**
__粗体文字__
**粗体 文字**  【正确示例，符号与文字连接部分无空格，文字内部允许有空格】
** 不是粗体文字**  【错误示例】
**不是粗体文字 **  【错误示例】
这处是**粗体文字**和***粗体＋斜体文字***
这处是*_粗体文字吗_*  【错误示例】
这处是_*粗体文字吗*_  【错误示例】
这处是**_粗体＋斜体文字吗_**  【错误示例】
这处是_**粗体＋斜体文字吗**_  【错误示例】
这处是__粗体文字吗__，这处是___粗体＋斜体文字吗___  【错误示例】
```

### 3.2.2. 实现效果

> **粗体文字**
>
> __粗体文字__
>
> **粗体 文字**  【正确示例，符号与文字连接部分无空格，文字内部允许有空格】
>
> ** 不是粗体文字**  【错误示例】
>
> **不是粗体文字 **  【错误示例】
>
> 这处是**粗体文字**和***粗体＋斜体文字***
>
> 这处是*_粗体文字吗_*  【错误示例】
>
> 这处是_*粗体文字吗*_  【错误示例】
>
> 这处是**_粗体＋斜体文字吗_**  【错误示例】
>
> 这处是_**粗体＋斜体文字吗**_  【错误示例】
>
> 这行是__粗体文字吗__，这处是___粗体＋斜体文字吗___  【错误示例】

### 3.3. 加删除线文字

加删除线文字用两对波浪号```~~```将待加删除线的文字包围来定义。

### 3.3.1. 示例代码

```
~~删除线文字~~
~~*删除线＋斜体文字*~~
**~~_删除线+粗体＋斜体文字_~~**
这处是**~~_删除线+粗体＋斜体文字吗_~~**  【错误示例】
这处是***~~删除线+粗体＋斜体文字吗~~***  【错误示例】
这处是~~***删除线+粗体＋斜体文字吗***~~  【错误示例】
```

### 3.3.2. 实现效果

> ~~删除线文字~~
>
> ~~*删除线＋斜体文字*~~
>
> **~~_删除线+粗体＋斜体文字_~~**
>
> 这处是**~~_删除线+粗体＋斜体文字吗_~~**  【错误示例】
>
> 这处是***~~删除线+粗体＋斜体文字吗~~***  【错误示例】
>
> 这处是~~***删除线+粗体＋斜体文字吗***~~  【错误示例】
